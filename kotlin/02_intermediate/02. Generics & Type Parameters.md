# Generics & Type Parameters

## Overview

Generics allow you to write type-safe code that works with different types. Kotlin's generics system includes variance modifiers (in, out) and reified type parameters for more powerful generic programming.

## Deep Explanation

### Basic Generics

#### Generic Classes

```kotlin
class Box<T>(var content: T) {
    fun get(): T = content
    fun set(value: T) {
        content = value
    }
}

val intBox = Box(42)
val stringBox = Box("Hello")
```

#### Generic Functions

```kotlin
fun <T> singletonList(item: T): List<T> {
    return listOf(item)
}

val list = singletonList("Kotlin")
```

### Variance

#### Invariance (Default)

```kotlin
class Box<T>

val intBox: Box<Int> = Box()
// val numberBox: Box<Number> = intBox  // ❌ Error: invariant
```

#### Covariance (`out`)

Producer - can only return T, cannot consume:

```kotlin
interface Producer<out T> {
    fun produce(): T
}

val intProducer: Producer<Int> = object : Producer<Int> {
    override fun produce() = 42
}
val numberProducer: Producer<Number> = intProducer  // ✅ OK
```

#### Contravariance (`in`)

Consumer - can only consume T, cannot return:

```kotlin
interface Consumer<in T> {
    fun consume(item: T)
}

val numberConsumer: Consumer<Number> = object : Consumer<Number> {
    override fun consume(item: Number) {}
}
val intConsumer: Consumer<Int> = numberConsumer  // ✅ OK
```

### Reified Type Parameters

Only works with inline functions:

```kotlin
inline fun <reified T> isInstanceOf(value: Any): Boolean {
    return value is T
}

println(isInstanceOf<String>("Hello"))  // true
```

### Type Projections

#### Star Projections

```kotlin
fun printList(list: List<*>) {
    for (item in list) {
        println(item)
    }
}
```

### Examples

#### Example 1: Generic Repository

```kotlin
interface Repository<T> {
    fun save(item: T)
    fun findById(id: Int): T?
    fun findAll(): List<T>
}
```

#### Example 2: Type-Safe Builder

```kotlin
inline fun <reified T> create(): T {
    return T::class.java.newInstance()
}
```

## Summary

Generics provide type safety and code reuse:
- Variance modifiers (`in`, `out`) control subtyping
- Reified types enable runtime type checks
- Type projections allow working with unknown types
