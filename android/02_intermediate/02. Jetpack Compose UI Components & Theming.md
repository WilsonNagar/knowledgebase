# Jetpack Compose UI Components & Theming

## Overview

Jetpack Compose provides a comprehensive set of Material Design 3 components and powerful theming capabilities. This guide covers Material components, custom composables, theming (light/dark mode), typography, icons, images, animations, and best practices for building beautiful, accessible Android UIs.


## Theming

### Color Scheme

```kotlin
val DarkColorScheme = darkColorScheme(
    primary = Purple80,
    secondary = PurpleGrey80,
    tertiary = Pink80
)

val LightColorScheme = lightColorScheme(
    primary = Purple40,
    secondary = PurpleGrey40,
    tertiary = Pink40
)
```

### Material Theme Setup

```kotlin
@Composable
fun MyApp(
    darkTheme: Boolean = isSystemInDarkTheme(),
    dynamicColor: Boolean = true
) {
    val colorScheme = when {
        dynamicColor && Build.VERSION.SDK_INT >= Build.VERSION_CODES.S -> {
            val context = LocalContext.current
            if (darkTheme) dynamicDarkColorScheme(context)
            else dynamicLightColorScheme(context)
        }
        darkTheme -> DarkColorScheme
        else -> LightColorScheme
    }
    
    MaterialTheme(
        colorScheme = colorScheme,
        typography = Typography,
        content = content
    )
}
```

### Using Theme Colors

## Animations

### AnimatedVisibility

```kotlin
var visible by remember { mutableStateOf(false) }

AnimatedVisibility(visible = visible) {
    Text("Animated Text")
}
```

### Crossfade

```kotlin
var currentScreen by remember { mutableStateOf("Screen1") }

Crossfade(targetState = currentScreen) { screen ->
    when (screen) {
        "Screen1" -> Screen1()
        "Screen2" -> Screen2()
    }
}
```

### Animated Content

```kotlin
var count by remember { mutableStateOf(0) }

AnimatedContent(
    targetState = count,
    transitionSpec = {
        slideInVertically { height -> height } + fadeIn() togetherWith
        slideOutVertically { height -> -height } + fadeOut()
    }
) { targetCount ->
    Text("$targetCount")
}
```

### Transition Animations

```kotlin
var expanded by remember { mutableStateOf(false) }

val transition = updateTransition(targetState = expanded, label = "expansion")

val height by transition.animateDp(label = "height") { isExpanded ->
    if (isExpanded) 200.dp else 0.dp
}

Box(
    modifier = Modifier
        .height(height)
        .fillMaxWidth()
) {
    // Content
}
```

## Custom Composables

### Reusable Card Component

```kotlin
@Composable
fun CustomCard(
    title: String,
    content: String,
    modifier: Modifier = Modifier,
    onClick: () -> Unit = {}
) {
    Card(
        modifier = modifier
            .fillMaxWidth()
            .clickable { onClick() },
        elevation = CardDefaults.cardElevation(defaultElevation = 4.dp)
    ) {
        Column(
            modifier = Modifier.padding(16.dp)
        ) {
            Text(
                text = title,
                style = MaterialTheme.typography.titleLarge
            )
            Spacer(modifier = Modifier.height(8.dp))
            Text(
                text = content,
                style = MaterialTheme.typography.bodyMedium
            )
        }
    }
}
```

## Best Practices

### Practice 1: Use Material Components

```kotlin
// GOOD: Use Material components
Button(onClick = { }) {
    Text("Click")
}

// BAD: Custom button when Material exists
Box(
    modifier = Modifier
        .background(Color.Blue)
        .clickable { }
) {
    Text("Click")
}
```

### Practice 2: Follow Material Design Guidelines

```kotlin
// GOOD: Follow spacing guidelines
Column(
    modifier = Modifier.padding(16.dp),
    verticalArrangement = Arrangement.spacedBy(8.dp)
) {
    Text("Item 1")
    Text("Item 2")
}
```

### Practice 3: Use Theme Colors

```kotlin
// GOOD: Use theme colors
Text(
    text = "Text",
    color = MaterialTheme.colorScheme.onSurface
)

// BAD: Hard-coded colors
Text(
    text = "Text",
    color = Color.Black // Don't do this
)
```

### Practice 4: Provide Content Descriptions

```kotlin
// GOOD: Content description for accessibility
Icon(
    imageVector = Icons.Default.Favorite,
    contentDescription = "Favorite"
)

// BAD: Missing content description
Icon(imageVector = Icons.Default.Favorite)
```

### Practice 5: Use Modifiers Consistently

```kotlin
// GOOD: Consistent modifier usage
Card(
    modifier = Modifier
        .fillMaxWidth()
        .padding(16.dp)
        .clickable { }
) {
    // Content
}
```

## Real-World Examples

### Complete Screen Example

```kotlin
@Composable
fun UserProfileScreen(
    user: User,
    onEditClick: () -> Unit
) {
    Scaffold(
        topBar = {
            TopAppBar(
                title = { Text("Profile") },
                actions = {
                    IconButton(onClick = onEditClick) {
                        Icon(Icons.Default.Edit, "Edit")
                    }
                }
            )
        }
    ) { padding ->
        Column(
            modifier = Modifier
                .fillMaxSize()
                .padding(padding)
                .padding(16.dp),
            horizontalAlignment = Alignment.CenterHorizontally
        ) {
            AsyncImage(
                model = user.avatarUrl,
                contentDescription = "Profile Picture",
                modifier = Modifier
                    .size(120.dp)
                    .clip(CircleShape),
                contentScale = ContentScale.Crop
            )
            
            Spacer(modifier = Modifier.height(16.dp))
            
            Text(
                text = user.name,
                style = MaterialTheme.typography.headlineMedium
            )
            
            Spacer(modifier = Modifier.height(8.dp))
            
            Text(
                text = user.email,
                style = MaterialTheme.typography.bodyMedium,
                color = MaterialTheme.colorScheme.onSurfaceVariant
            )
            
            Spacer(modifier = Modifier.height(24.dp))
            
            Card(
                modifier = Modifier.fillMaxWidth()
            ) {
                Column(
                    modifier = Modifier.padding(16.dp)
                ) {
                    Text(
                        text = "About",
                        style = MaterialTheme.typography.titleMedium
                    )
                    Spacer(modifier = Modifier.height(8.dp))
                    Text(
                        text = user.bio,
                        style = MaterialTheme.typography.bodyMedium
                    )
                }
            }
        }
    }
}
```

## Common Pitfalls

### Problem: Not Using Theme Colors

```kotlin
// BAD: Hard-coded colors
Text(text = "Text", color = Color.Black)

// GOOD: Use theme colors
Text(
    text = "Text",
    color = MaterialTheme.colorScheme.onSurface
)
```

### Problem: Missing Content Descriptions

```kotlin
// BAD: No content description
Icon(imageVector = Icons.Default.Favorite)

// GOOD: Provide content description
Icon(
    imageVector = Icons.Default.Favorite,
    contentDescription = "Favorite"
)
```

## Quiz

1. What is Material Design 3?
   - **A)** Old design system
   - **B)** Google's latest design system for Android
   - **C)** iOS design system
   - **D)** Web-only design system

2. How do you enable dark mode in Compose?
   - **A)** Use DarkColorScheme in MaterialTheme
   - **B)** Change system settings only
   - **C)** Not possible
   - **D)** Use separate theme file

3. What should you always provide for icons?
   - **A)** Size
   - **B)** Color
   - **C)** Content description
   - **D)** Nothing

**Answers:**
1. **B** - Material Design 3 is Google's latest design system, fully supported in Jetpack Compose
2. **A** - Use DarkColorScheme in MaterialTheme and switch based on system or user preference
3. **C** - Always provide content descriptions for accessibility (screen readers)
