# Accessibility in Android

## Overview

Accessibility ensures your Android app is usable by everyone, including users with disabilities. This comprehensive guide covers content descriptions, TalkBack support, semantic properties in Compose, accessibility testing, touch targets, text scaling, color contrast, and best practices for building inclusive Android applications.

## What is Accessibility?

### Definition

**Accessibility (a11y)**: Making apps usable by everyone

**Including**:
- **Visual Impairments**: Screen readers, magnification
- **Motor Impairments**: Large touch targets, voice control
- **Hearing Impairments**: Visual indicators, captions
- **Cognitive Impairments**: Clear labels, simple navigation

### Why Accessibility Matters

**Benefits**:
- **Inclusive**: Reach more users
- **Legal**: Compliance with accessibility laws
- **Better UX**: Improves experience for all users
- **Market**: Larger user base

## Content Descriptions

### What are Content Descriptions?

**Content Description**: Text description of UI element

**Purpose**: Screen readers announce this text

### Adding Content Descriptions

**In Views**:
```kotlin
imageView.contentDescription = "User profile picture"
button.contentDescription = "Submit form"
```

**In Compose**:
```kotlin
Image(
    painter = painterResource(id = R.drawable.icon),
    contentDescription = "User profile picture"
)

IconButton(onClick = { }) {
    Icon(
        imageVector = Icons.Default.Favorite,
        contentDescription = "Add to favorites"
    )
}
```

### When to Provide Descriptions

**Always Provide**:
- Icons without text
- Images (decorative or informative)
- Buttons without text labels

**Don't Provide**:
- Text views (they have their own text)
- Buttons with text labels

### Descriptive vs Decorative

**Descriptive**:
```kotlin
Image(
    painter = painterResource(id = R.drawable.product),
    contentDescription = "Red t-shirt, size medium"
)
```

**Decorative**:
```kotlin
Image(
    painter = painterResource(id = R.drawable.decorative_border),
    contentDescription = null // Decorative, not important
)
```

## TalkBack Support

### What is TalkBack?

**TalkBack**: Android screen reader

**Purpose**: Reads screen content aloud

### Testing with TalkBack

**Enable TalkBack**:
1. Settings → Accessibility → TalkBack
2. Turn on TalkBack
3. Navigate with gestures

**Gestures**:
- **Swipe Right**: Next element
- **Swipe Left**: Previous element
- **Double Tap**: Activate
- **Swipe Up/Down**: Navigate by element type

### Making Views TalkBack-Friendly

```kotlin
// GOOD: Proper content description
button.contentDescription = "Submit form"

// BAD: Missing or poor description
button.contentDescription = "Button" // Too generic
```

### Custom Actions

```kotlin
ViewCompat.addAccessibilityAction(view, "Custom Action") {
    // Perform action
    true
}
```

## Compose Accessibility

### Semantic Properties

```kotlin
Text(
    text = "Hello",
    modifier = Modifier.semantics {
        contentDescription = "Greeting message"
        role = Role.Text
    }
)
```

### Clickable Semantics

```kotlin
Box(
    modifier = Modifier
        .clickable { }
        .semantics {
            role = Role.Button
            contentDescription = "Submit button"
        }
) {
    Text("Submit")
}
```

### Heading Semantics

```kotlin
Text(
    text = "Section Title",
    style = MaterialTheme.typography.headlineMedium,
    modifier = Modifier.semantics {
        heading()
    }
)
```

### State Descriptions

```kotlin
var checked by remember { mutableStateOf(false) }

Checkbox(
    checked = checked,
    onCheckedChange = { checked = it },
    modifier = Modifier.semantics {
        stateDescription = if (checked) "Selected" else "Not selected"
    }
)
```

### Custom Actions in Compose

```kotlin
Box(
    modifier = Modifier.semantics {
        customActions = listOf(
            CustomAccessibilityAction("Delete") {
                deleteItem()
                true
            }
        )
    }
) {
    // Content
}
```

## Touch Targets

### Minimum Size

**Guideline**: Minimum 48dp × 48dp touch target

**Why**: Easier to tap for users with motor impairments

```kotlin
// GOOD: Minimum 48dp
Button(
    onClick = { },
    modifier = Modifier.size(48.dp)
) {
    Icon(Icons.Default.Add, "Add")
}

// BAD: Too small
IconButton(
    onClick = { },
    modifier = Modifier.size(24.dp) // Too small!
) {
    Icon(Icons.Default.Add, "Add")
}
```

### Spacing

**Guideline**: Minimum 8dp spacing between touch targets

```kotlin
Row(
    horizontalArrangement = Arrangement.spacedBy(8.dp)
) {
    Button(onClick = { }) { Text("Button 1") }
    Button(onClick = { }) { Text("Button 2") }
}
```

## Text Scaling

### Support Text Scaling

**Automatic**: Text views scale automatically

**Test**: Settings → Display → Font size

### Using sp Units

```kotlin
// GOOD: Use sp for text
Text(
    text = "Hello",
    fontSize = 16.sp // Scales with user settings
)

// BAD: Use dp for text
Text(
    text = "Hello",
    fontSize = 16.dp // Doesn't scale!
)
```

### Handling Large Text

```kotlin
Text(
    text = longText,
    maxLines = 3,
    overflow = TextOverflow.Ellipsis,
    modifier = Modifier.semantics {
        // Provide full text for screen readers
        contentDescription = longText
    }
)
```

## Color Contrast

### Contrast Ratios

**Guideline**: 
- **Normal Text**: 4.5:1 contrast ratio
- **Large Text**: 3:1 contrast ratio

**Purpose**: Readable for users with low vision

### Testing Contrast

**Tools**:
- Android Studio Accessibility Scanner
- Online contrast checkers

### Don't Rely on Color Alone

```kotlin
// BAD: Color only
Text(
    text = "Error",
    color = Color.Red
)

// GOOD: Color + icon/text
Row {
    Icon(
        imageVector = Icons.Default.Error,
        contentDescription = "Error",
        tint = Color.Red
    )
    Text(
        text = "Error",
        color = Color.Red
    )
}
```

## Accessibility Testing

### Android Studio Accessibility Scanner

**Usage**:
1. Run app on device/emulator
2. Tools → Accessibility Scanner
3. Scan for issues
4. Review recommendations

### Manual Testing

**Steps**:
1. Enable TalkBack
2. Navigate through app
3. Verify all content is accessible
4. Test with different font sizes
5. Test with different display sizes

### Automated Testing

```kotlin
@Test
fun testAccessibility() {
    val node = composeTestRule.onNodeWithContentDescription("Submit")
    node.assertExists()
    node.assertHasClickAction()
}
```

## Real-World Examples

### Accessible Button

```kotlin
@Composable
fun AccessibleButton(
    text: String,
    onClick: () -> Unit,
    modifier: Modifier = Modifier
) {
    Button(
        onClick = onClick,
        modifier = modifier
            .minimumInteractiveComponentSize()
            .semantics {
                role = Role.Button
                contentDescription = text
            }
    ) {
        Text(text)
    }
}
```

### Accessible Image List

```kotlin
@Composable
fun ImageList(images: List<ImageItem>) {
    LazyColumn {
        items(images) { image ->
            Card(
                modifier = Modifier
                    .fillMaxWidth()
                    .clickable { /* Handle click */ }
                    .semantics {
                        contentDescription = "${image.title}. ${image.description}"
                        role = Role.Button
                    }
            ) {
                Column {
                    AsyncImage(
                        model = image.url,
                        contentDescription = image.description,
                        modifier = Modifier.fillMaxWidth()
                    )
                    Text(
                        text = image.title,
                        style = MaterialTheme.typography.titleMedium
                    )
                }
            }
        }
    }
}
```

## Common Pitfalls

### Problem: Missing Content Descriptions

```kotlin
// BAD: No content description
IconButton(onClick = { }) {
    Icon(Icons.Default.Favorite)
}

// GOOD: Content description
IconButton(onClick = { }) {
    Icon(
        imageVector = Icons.Default.Favorite,
        contentDescription = "Add to favorites"
    )
}
```

### Problem: Too Small Touch Targets

```kotlin
// BAD: Too small
IconButton(
    onClick = { },
    modifier = Modifier.size(24.dp)
)

// GOOD: Minimum size
IconButton(
    onClick = { },
    modifier = Modifier.size(48.dp)
)
```

## Quiz

1. What is the minimum touch target size?
   - **A)** 24dp × 24dp
   - **B)** 48dp × 48dp
   - **C)** 64dp × 64dp
   - **D)** No minimum

2. What should you always provide for icons?
   - **A)** Size
   - **B)** Color
   - **C)** Content description
   - **D)** Nothing

3. What unit should you use for text to support scaling?
   - **A)** dp
   - **B)** sp
   - **C)** px
   - **D)** pt

**Answers:**
1. **B** - Minimum touch target size is 48dp × 48dp for accessibility
2. **C** - Always provide content descriptions for icons so screen readers can announce them
3. **B** - Use sp (scale-independent pixels) for text to support user font size preferences
