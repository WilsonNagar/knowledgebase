# Automotive Infotainment Development

## Overview

Android Automotive OS and Android Auto enable developers to create apps for car infotainment systems. This guide covers developing apps for automotive environments, handling car-specific constraints, and creating safe, distraction-free user experiences.

## Deep Explanation

### Android Auto vs Android Automotive OS

**Android Auto:**
- Phone projection to car screen
- Apps run on phone, display on car
- Requires phone connection

**Android Automotive OS:**
- Native Android OS in car
- Apps run directly on car system
- No phone required

### Automotive Constraints

- **Safety First**: Minimize driver distraction
- **Large Touch Targets**: Easy to tap while driving
- **Voice Commands**: Hands-free interaction
- **Limited Resources**: Optimize for car hardware

## Common Pitfalls

### Problem: Small touch targets

**BAD:**
```kotlin
// Too small for car use
Button(width = 40.dp, height = 40.dp)
```

**GOOD:**
```kotlin
// Minimum 48dp, preferably larger
Button(minWidth = 60.dp, minHeight = 60.dp)
```
