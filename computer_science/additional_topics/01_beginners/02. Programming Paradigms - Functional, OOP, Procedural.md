# Programming Paradigms - Functional, OOP, Procedural

## Overview

## What are Programming Paradigms?

### Definition

**Programming Paradigm**: Style of programming

**Purpose**: 
```
Organize code
Solve problems
Express solutions
```

**Types**: 
- **Procedural**: Step-by-step procedures
- **Object-Oriented**: Objects and classes
- **Functional**: Functions and immutability
- **Declarative**: What, not how

**Use**: Different problem domains

### Paradigm Characteristics

**1. Code Organization**: 
```
How code is structured
```

**2. Data Handling**: 
```
How data is managed
```

**3. Control Flow**: 
```
How execution flows
```

**4. Abstraction**: 
```
Level of abstraction
```

## Procedural Programming

### What is Procedural Programming?

**Procedural**: 
```
Step-by-step procedures
Functions/subroutines
```

**Characteristics**: 
- **Functions**: Reusable functions
- **Sequential**: Sequential execution
- **State**: Mutable state
- **Top-Down**: Top-down design

**Languages**: C, Pascal, Fortran

### Procedural Example

**Example**:
```c
int add(int a, int b) {
    return a + b;
}

int multiply(int a, int b) {
    return a * b;
}

int main() {
    int x = 5;
    int y = 3;
    int sum = add(x, y);
    int product = multiply(x, y);
    return 0;
}
```

**Structure**: Functions operate on data

## Object-Oriented Programming

### What is OOP?

**OOP**: 
```
Objects encapsulate data and behavior
Classes define objects
```

**Principles**: 
- **Encapsulation**: Hide implementation
- **Inheritance**: Reuse code
- **Polymorphism**: Multiple forms
- **Abstraction**: Abstract concepts

**Languages**: Java, C++, Python, C#

### OOP Example

**Example**:
```java
class Animal {
    private String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    public void speak() {
        System.out.println("Animal speaks");
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void speak() {
        System.out.println("Woof!");
    }
}
```

**Structure**: Objects with state and behavior

## Functional Programming

### What is Functional Programming?

**Functional**: 
```
Functions as first-class citizens
Immutability
Pure functions
```

**Characteristics**: 
- **Immutability**: No mutable state
- **Pure Functions**: No side effects
- **Higher-Order**: Functions as values
- **Recursion**: Recursive thinking

**Languages**: Haskell, Lisp, Scala, F#

### Functional Example

**Example**:
```haskell
-- Pure function
add :: Int -> Int -> Int
add x y = x + y

-- Higher-order function
map :: (a -> b) -> [a] -> [b]
map f [] = []
map f (x:xs) = f x : map f xs

-- Usage
double = map (* 2)
result = double [1, 2, 3]  -- [2, 4, 6]
```

**Structure**: Functions transform data

## Paradigm Comparison

### Comparison Table

| Aspect | Procedural | OOP | Functional |
|--------|-----------|-----|------------|
| **Focus** | Procedures | Objects | Functions |
| **State** | Mutable | Mutable | Immutable |
| **Reusability** | Functions | Classes | Functions |
| **Complexity** | Low-Medium | Medium-High | Medium |
| **Performance** | High | Medium | Variable |

### When to Use Each

**Procedural**: 
- **Simple**: Simple programs
- **Performance**: Performance critical
- **System**: System programming

**OOP**: 
- **Complex**: Complex systems
- **Modeling**: Real-world modeling
- **Reusability**: Code reuse

**Functional**: 
- **Concurrency**: Concurrent systems
- **Data Processing**: Data transformation
- **Correctness**: Mathematical correctness

## Quiz

1. What is procedural programming?
   - **A)** Object-based
   - **B)** Step-by-step procedures and functions
   - **C)** Functional only
   - **D)** Declarative

2. What are OOP principles?
   - **A)** Functions only
   - **B)** Encapsulation, inheritance, polymorphism, abstraction
   - **C)** Immutability
   - **D)** Recursion

3. What is functional programming?
   - **A)** Objects only
   - **B)** Functions as first-class, immutability, pure functions
   - **C)** Procedures
   - **D)** Classes

**Answers:**
1. **B** - Procedural programming organizes code as step-by-step procedures and reusable functions
2. **B** - OOP principles include encapsulation (data hiding), inheritance (code reuse), polymorphism (multiple forms), and abstraction
3. **B** - Functional programming emphasizes functions as first-class citizens, immutability, and pure functions without side effects
