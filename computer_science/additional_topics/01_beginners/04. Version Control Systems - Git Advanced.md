## Git Internals

### Git Object Model

**Git Objects**: 
- **Blobs**: File contents
- **Trees**: Directory structure
- **Commits**: Snapshots
- **Tags**: Named references

**Storage**: 
```
.git/objects/
SHA-1 hashes
```

**Structure**: 
```
Commit → Tree → Blobs
```

### Git References

**References**: 
- **HEAD**: Current branch
- **Branches**: Branch pointers
- **Tags**: Tag pointers
- **Remote**: Remote references

**Location**: 
```
.git/refs/
```

## Rebasing

### What is Rebasing?

**Rebase**: 
```
Reapply commits on top of another branch
Linear history
```

**vs Merge**: 
```
Merge: Creates merge commit
Rebase: Linear history
```

**Use**: Clean history

### Interactive Rebasing

**Interactive Rebase**: 
```bash
git rebase -i HEAD~3
```

**Actions**: 
- **pick**: Use commit
- **reword**: Change message
- **edit**: Modify commit
- **squash**: Combine commits
- **drop**: Remove commit

**Use**: Clean up history

### Rebase Example

**Before**:
```
A---B---C  master
     \
      D---E  feature
```

**After rebase**:
```bash
git checkout feature
git rebase master
```

```
A---B---C  master
         \
          D'---E'  feature
```

## Cherry-Picking

### What is Cherry-Picking?

**Cherry-Pick**: 
```
Apply specific commit to current branch
```

**Use**: 
- **Backport**: Backport fixes
- **Selective**: Select commits
- **Hotfix**: Apply hotfixes

**Command**: 
```bash
git cherry-pick <commit-hash>
```

### Cherry-Pick Example

**Example**: 
```bash
# Apply commit from other branch
git cherry-pick abc123

# Apply multiple commits
git cherry-pick abc123 def456

# Apply range
git cherry-pick abc123..def456
```

## Git Hooks

### What are Git Hooks?

**Git Hooks**: 
```
Scripts run at Git events
```

**Types**: 
- **pre-commit**: Before commit
- **post-commit**: After commit
- **pre-push**: Before push
- **pre-rebase**: Before rebase

**Location**: 
```
.git/hooks/
```

### Hook Example

**pre-commit Hook**: 
```bash
#!/bin/sh
# Run tests before commit
npm test
if [ $? -ne 0 ]; then
    echo "Tests failed"
    exit 1
fi
```

**Use**: Enforce quality

**GitHub Flow**:

**Good Messages**:

```
Present tense
```

**Types**: 
- **feat**: New feature
- **fix**: Bug fix

- **refactor**: Refactoring

**Conventions**:

### Practice 3: Regular Commits

**Regular**:

```
Commit often
Small, logical commits
```

## Common Pitfalls

### Problem: Force Push

```c
// BAD: Force push to shared branch
// Rewrite history

// GOOD: Only force push to personal branches
// Or coordinate with team
```

### Problem: Large Commits

```c
// BAD: Huge commits
// Hard to review

// GOOD: Small, logical commits
// Easier to understand
```

## Quiz

1. What is rebasing?
   - **A)** Merging branches
   - **B)** Reapplying commits on top of another branch for linear history
   - **C)** Deleting commits
   - **D)** Creating branches

2. What is cherry-picking?
   - **A)** Creating branches
   - **B)** Applying specific commits to current branch
   - **C)** Merging branches
   - **D)** Deleting commits

3. What are Git hooks?
   - **A)** Branch names
   - **B)** Scripts run at Git events (pre-commit, post-commit, etc.)
   - **C)** Commit messages
   - **D)** Remote repositories

**Answers:**
1. **B** - Rebasing replays commits on top of another branch, creating linear history instead of merge commits
2. **B** - Cherry-picking applies specific commits from one branch to another, useful for backporting fixes
3. **B** - Git hooks are scripts that run automatically at Git events like pre-commit, post-commit, pre-push
