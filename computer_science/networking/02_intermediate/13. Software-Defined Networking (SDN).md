# Software-Defined Networking (SDN)

## Overview

Software-Defined Networking (SDN) separates the control plane from the data plane, enabling centralized network control and programmability. Understanding SDN architecture, OpenFlow, SDN controllers, and network programmability is essential for modern network architecture and automation.

## What is SDN?

### Definition

**SDN**: Software-Defined Networking

**Key Concept**: Separate control and data planes

**Control Plane**: Centralized, programmable

**Data Plane**: Forwarding only

**Benefit**: Centralized control, programmability

## OpenFlow Protocol

### What is OpenFlow?

**OpenFlow**: SDN protocol

**Purpose**: Control forwarding in switches

**Method**: Controller sends flow rules to switches

**Standard**: Open Networking Foundation

### OpenFlow Flow Table

**Flow Table**: Match-action table

**Match**: Match packet fields

**Actions**: Forward, drop, modify

**Example**:
```
Match: src_ip=10.0.0.1, dst_port=80
Action: Forward to port 2
```

### OpenFlow Messages

**1. Controller-to-Switch**:
```
Configure switch
Modify flow table
```

**2. Asynchronous**:
```
Switch to controller
Packet-in, flow removed
```

**3. Symmetric**:
```
Hello, echo
```

## Quiz

1. What is SDN?
   - **A)** Software network
   - **B)** Software-Defined Networking - separating control and data planes for centralized control
   - **C)** Secure network
   - **D)** Simple network

2. What is OpenFlow?
   - **A)** Open protocol
   - **B)** SDN protocol for controlling forwarding in switches
   - **C)** Open file
   - **D)** Open framework

3. What is the main benefit of SDN?
   - **A)** Faster hardware
   - **B)** Centralized control and programmability
   - **C)** More devices
   - **D)** Simpler hardware

**Answers:**
1. **B** - SDN (Software-Defined Networking) separates the control plane (network logic) from the data plane (forwarding), enabling centralized control and programmability
2. **B** - OpenFlow is a southbound protocol that allows SDN controllers to communicate with switches, installing flow rules that control packet forwarding
3. **B** - The main benefits of SDN are centralized control (easier management) and programmability (ability to program network behavior), enabling automation and rapid innovation
