# Blockchain Fundamentals

## Overview

Blockchain is a distributed ledger technology that maintains a continuously growing list of records secured by cryptography. Understanding blockchain architecture, consensus mechanisms, smart contracts, and distributed ledger principles is essential for understanding cryptocurrencies and blockchain applications.

## What is Blockchain?

### Definition

**Blockchain**: Distributed ledger

**Structure**: Chain of blocks

**Properties**: 
- **Immutable**: Immutable records
- **Distributed**: Distributed across nodes
- **Cryptographic**: Cryptographically secured
- **Consensus**: Consensus-based

**Use**: Cryptocurrencies, distributed applications

### Blockchain Characteristics

**1. Decentralization**:
```
No central authority
```

**2. Immutability**:
```
Records cannot be changed
```

**3. Transparency**:
```
Public ledger
```

**4. Security**:
```
Cryptographically secured
```

## Blockchain Architecture

### Block Structure

**Block**: 
- **Header**: Block header
- **Transactions**: List of transactions
- **Hash**: Block hash

**Header**: 
- **Previous Hash**: Previous block hash
- **Merkle Root**: Merkle root
- **Timestamp**: Timestamp
- **Nonce**: Nonce (PoW)

### Blockchain Structure

**Chain**:
```
Block 1 → Block 2 → Block 3 → ...
```

**Each Block**: 
- **Previous Hash**: Links to previous
- **Transactions**: Contains transactions
- **Hash**: Cryptographic hash

**Immutability**: Changing block breaks chain

## Cryptographic Hashing

### Hash Functions

**Hash Function**: One-way function

**Properties**: 
- **Deterministic**: Same input → same output
- **Fast**: Fast computation
- **One-Way**: Cannot reverse
- **Avalanche**: Small change → big change

**Use**: Block hashing, Merkle trees

### Merkle Tree

**Merkle Tree**: Tree of hashes

**Structure**:
```
        Root Hash
       /        \
   Hash1-2    Hash3-4
   /    \      /    \
 Hash1 Hash2 Hash3 Hash4
```

**Benefit**: Efficient verification

## Consensus Mechanisms

### What is Consensus?

**Consensus**: Agreement on state

**Purpose**: Agree on valid blocks

**Challenges**: 
- **Byzantine Faults**: Byzantine failures
- **Network**: Network delays
- **Scalability**: Scalability

**Mechanisms**: 
- **Proof of Work**: PoW
- **Proof of Stake**: PoS
- **PBFT**: Practical Byzantine Fault Tolerance

## Proof of Work

### What is Proof of Work?

**Proof of Work**: Computational work

**Process**: 
```
1. Collect transactions
2. Create block
3. Find nonce (hash < target)
4. Broadcast block
5. Others verify
```

**Difficulty**: Adjustable difficulty

**Security**: High computational cost

### PoW Example

**Mining**:
```
Block: transactions + previous_hash + nonce
Hash = SHA256(block)
If hash < target: Valid block
Else: Try different nonce
```

**Difficulty**: Target determines difficulty

**Energy**: High energy consumption

## Proof of Stake

### What is Proof of Stake?

**Proof of Stake**: Stake-based consensus

**Process**: 
```
1. Validators stake coins
2. Validator selected (based on stake)
3. Validator creates block
4. Others validate
```

**Energy**: Lower energy consumption

**Security**: Economic security

### PoS Benefits

**1. Energy Efficient**:
```
Lower energy consumption
```

**2. Faster**:
```
Faster block creation
```

**3. Scalable**:
```
Better scalability
```

**4. Economic Security**:
```
Economic incentives
```

## Smart Contracts

### What are Smart Contracts?

**Smart Contracts**: Self-executing contracts

**Purpose**: Automate agreements

**Execution**: Execute on blockchain

**Immutability**: Cannot be changed

**Use**: DeFi, DApps, automation

### Smart Contract Example

**Simple Contract**:
```solidity
contract SimpleStorage {
    uint256 public value;
    
    function set(uint256 _value) public {
        value = _value;
    }
    
    function get() public view returns (uint256) {
        return value;
    }
}
```

**Execution**: Execute on blockchain

**Gas**: Pay gas for execution
