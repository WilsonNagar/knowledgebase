# Penetration Testing Fundamentals

## Overview

Penetration testing simulates real-world attacks to identify security vulnerabilities. Understanding penetration testing methodologies, tools, techniques, and reporting is essential for security professionals and organizations seeking to improve their security posture.

## What is Penetration Testing?

### Definition

**Penetration Testing**: Authorized security testing

**Purpose**: Identify vulnerabilities

**Method**: Simulate real attacks

**Benefit**: Improve security

### Penetration Testing vs Vulnerability Assessment

**Vulnerability Assessment**:
- **Scope**: Identify vulnerabilities
- **Depth**: Surface-level
- **Method**: Automated scanning

**Penetration Testing**:
- **Scope**: Exploit vulnerabilities
- **Depth**: Deep analysis
- **Method**: Manual + automated

## Reconnaissance

### What is Reconnaissance?

**Reconnaissance**: Information gathering

**Purpose**: Gather target information

**Types**:
- **Passive**: No direct interaction
- **Active**: Direct interaction

### Passive Reconnaissance

**Methods**:
- **OSINT**: Open source intelligence
- **Google Dorking**: Advanced searches
- **Social Media**: Social media analysis
- **DNS**: DNS enumeration

**Tools**: 
- **theHarvester**: Email, subdomain discovery
- **Maltego**: Link analysis
- **Shodan**: Internet-connected device search

### Active Reconnaissance

**Methods**:
- **Port Scanning**: Scan ports
- **Service Detection**: Detect services
- **Banner Grabbing**: Grab banners
- **DNS Queries**: Query DNS

**Tools**:
- **Nmap**: Network scanner
- **Masscan**: Fast port scanner

## Scanning & Enumeration

### Port Scanning

**Purpose**: Discover open ports

**Techniques**:
- **TCP SYN**: Stealth scan
- **TCP Connect**: Full connection
- **UDP**: UDP scan
- **Stealth**: Avoid detection

**Nmap Examples**:
```bash
nmap -sS target.com        # SYN scan
nmap -sV target.com        # Version detection
nmap -sC target.com        # Default scripts
nmap -p- target.com        # All ports
```

### Service Enumeration

**Purpose**: Identify services

**Methods**:
- **Banner Grabbing**: Grab banners
- **Version Detection**: Detect versions
- **Service Probes**: Probe services

**Tools**:
- **Nmap**: Service detection
- **Netcat**: Banner grabbing

### Vulnerability Scanning

**Purpose**: Find vulnerabilities

**Tools**:
- **Nessus**: Commercial scanner
- **OpenVAS**: Open-source scanner
- **Nexpose**: Commercial scanner

## Quiz

1. What is penetration testing?
   - **A)** Vulnerability scanning
   - **B)** Authorized security testing that simulates real attacks to identify vulnerabilities
   - **C)** Security monitoring
   - **D)** Incident response

2. What is the difference between black box and white box testing?
   - **A)** No difference
   - **B)** Black box has no prior knowledge, white box has full knowledge
   - **C)** Black box is faster
   - **D)** White box is cheaper

3. What is Metasploit?
   - **A)** Vulnerability scanner
   - **B)** Exploitation framework for developing and executing exploits
   - **C)** SIEM system
   - **D)** Firewall

**Answers:**
1. **B** - Penetration testing is authorized security testing that simulates real-world attacks to identify and exploit vulnerabilities, helping organizations improve security
2. **B** - Black box testing simulates external attackers with no prior knowledge, while white box testing uses full knowledge of the system for comprehensive testing
3. **B** - Metasploit is an exploitation framework that provides exploits, payloads, and auxiliary modules for penetration testing and security research
