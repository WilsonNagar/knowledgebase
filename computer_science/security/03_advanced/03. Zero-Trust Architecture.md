# Zero-Trust Architecture

## Overview

Zero-Trust Architecture is a security model that assumes no implicit trust and requires verification for every access request. Understanding zero-trust principles, implementation strategies, identity and access management, and micro-segmentation is essential for modern security architectures.

## What is Zero-Trust?

### Definition

**Zero-Trust**: Never trust, always verify

**Traditional**: Trust internal network

**Zero-Trust**: Verify every request

**Principle**: Assume breach

### Zero-Trust Philosophy

**Core Tenet**: Never trust, always verify

**Assumptions**:
- **No Implicit Trust**: No automatic trust
- **Assume Breach**: Assume compromised
- **Verify Explicitly**: Verify every request
- **Least Privilege**: Minimal access

## Quiz

1. What is Zero-Trust?
   - **A)** Trust everything
   - **B)** Security model that never trusts and always verifies every access request
   - **C)** Trust internal network
   - **D)** Trust external network

2. What is micro-segmentation?
   - **A)** Large segments
   - **B)** Fine-grained network segmentation to isolate workloads
   - **C)** No segmentation
   - **D)** Simple segmentation

3. What is the core principle of Zero-Trust?
   - **A)** Trust but verify
   - **B)** Never trust, always verify
   - **C)** Trust internal
   - **D)** Trust external

**Answers:**
1. **B** - Zero-Trust is a security model that assumes no implicit trust and requires explicit verification for every access request, regardless of location
2. **B** - Micro-segmentation creates fine-grained network segments to isolate workloads and prevent lateral movement of attackers
3. **B** - The core principle is "never trust, always verify" - every access request must be authenticated, authorized, and continuously validated
